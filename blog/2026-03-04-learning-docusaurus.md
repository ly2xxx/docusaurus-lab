# Learning Docusaurus

Today I set up a Docusaurus lab to understand how it works.

<!-- truncate -->

## Why Learn Docusaurus?

Even though I decided to use **Jekyll Collections** for my project catalog, Docusaurus is still valuable for:

- 📚 **Project documentation** - API docs, tutorials
- 🌍 **Multi-language support** - i18n built-in
- 📦 **Versioning** - Multiple doc versions side-by-side
- ⚛️ **React integration** - Full React ecosystem

## Quick Impressions

### ✅ Pros
- Hot reload is fast
- MDX is powerful (React in Markdown!)
- Dark mode looks great
- Plugin ecosystem is rich

### ⚠️ Cons
- Heavyweight for simple sites
- Overkill for project showcases
- Build time can be slow with many pages

## What I'll Use It For

Not for my catalog, but perfect for:

1. **aidev documentation** - Multi-page tutorial
2. **RAG chatbot guides** - Code examples with tabs
3. **AI/ML tutorials** - Versioned content
4. **Blog + docs combo** - All-in-one site

## Key Takeaway

> **Right tool for the right job.**
>
> - Docusaurus = Documentation powerhouse  
> - Jekyll = Lightweight catalog/portfolio

Both have their place! 🎯
